# 팩트체크 체크리스트 템플릿

> 쇼츠 작업 시 **기획안 후 1차**, **대본 후 2차** 팩트체크에 사용.
> 카테고리별 체크 항목을 로드해 주장된 팩트를 체계적으로 검증할 것.
> 연동: `feedback_factcheck_workflow.md` (2회 규칙) + `workflow_shorts_planning.md` 의사결정 매트릭스 Step 2.5 / 3.5

---

## 팩트체크 실행 프로세스

### 1차 팩트체크 (기획안 → 대본 전)

1. **항목 추출**: 기획안에서 카테고리 체크리스트에 해당하는 모든 팩트 주장을 추출
2. **검증 계획 수립**: 어떤 항목을 어떤 출처로 확인할지 표로 정리
3. **사용자 승인 대기** (매트릭스 Step 2.5에 따라 필수)
4. **검증 실행**: WebSearch + 공식 출처 교차 조회
5. **결과 보고**: 표 형식 — `주장 / 검증 / 출처 / 신뢰도 / 판정(OK·수정·추가확인)`
6. 오류 발견 시 기획안 수정 후 대본 단계 진행

### 2차 팩트체크 (대본 완성 후)

1. **대본에서 추가·변형된 주장만** 좁게 검증 (기획안 대비 delta)
2. 자율 검증 실행 (매트릭스 Step 3.5)
3. **의심 지점 발견 시 수정 전 보고**. 확실한 오류는 자율 수정하되 기록에 남김
4. 결과 보고 → 최종 확정

## 출처 신뢰도 순위

| 등급 | 출처 | 비고 |
|---|---|---|
| S | 공식 사이트·공식 문서 (제조사, 정부, 학회) | 최우선 |
| A | 위키피디아 **1차 출처 인용부** (본문 주석 링크) | 영문 위키 우선, 한글 위키는 오역 빈발 |
| B | 위키피디아 본문 (인용 없는 부분) | 보조용 |
| C | 주요 언론 (현지 매체 우선) | 수치는 여러 매체 교차 확인 필요 |
| D | 커뮤니티·블로그·개인 위키 | 단독 근거로 사용 금지, 참고만 |

**주의**: 한글 위키만 보고 판단하지 말 것. 공학·역사·외국 인물은 영문 위키가 거의 항상 더 정확.

## 카테고리별 체크리스트

### 공학 / 산업·중장비
**서브카테고리 예**: 굴착기·크레인·선박·플랜트·중장비·토목 구조물

| 항목 | 확인 포인트 |
|---|---|
| 제조사 | 풀네임 + 합병·분사 이력 (예: ThyssenKrupp ↔ TAKRAF). 작업 시점 기준 |
| 제조/운용 연도 | 제작 연도 vs 투입 연도 vs 현재 가동 여부 구분 |
| 크기 | 전체 길이, 높이, 폭, 부분별 치수 (버킷·붐·휠 지름 등) |
| 무게 | 총 중량. 설비 단위와 혼동 주의 (short ton vs metric ton) |
| 동력 | 엔진 출력 / 전력 소비 / 연료 종류 |
| 생산량·작업량 | **단위 주의** (일/연, m³/톤, 1회/누적). 바거 293 채굴량 단위 오류 사례 |
| 운영 지역 | 국가, 광산·공장·항구 이름 (원어 표기 병기) |
| 특수 사양 | 버킷 용량, 적재량, 최고 속도, 지면 압력, 회전 반경 등 |
| 인력 | 승무원 수, 교대 방식 |
| 가격·건설비 | 제작 당시 vs 현재가치 환산 여부 |

**놓치기 쉬운 함정**:
- 제조사 이름이 합병·인수로 변경된 경우 (원 제조사 vs 현 소유사)
- 수치 단위 혼동 (ft vs m, lb vs kg, ton 종류)
- "세계 최대" 류 주장 → 어느 범주에서 최대인지 (지상/해상, 현역/전체)

---

### 공학 / 과학·자연
**서브카테고리 예**: 과학 현상, 자연 기록, 우주, 생물, 발명·발견

| 항목 | 확인 포인트 |
|---|---|
| 발견·발명 연도 | 최초 관측 vs 공식 발표 vs 명명 연도 구분 |
| 발견자·발명자 | 풀네임 + 소속 기관. 공동 연구면 모두 기재 |
| 위치 | 지역·좌표·서식지. 관측지와 서식지가 다를 수 있음 |
| 규모·수치 | 길이·무게·크기·속도. **허용 오차 범위** 확인 |
| 메커니즘·원리 | 비유 사용 시 오해 여지 확인 (예: "블랙홀이 빨아들인다" 같은 부정확한 표현) |
| 시대·기간 | 지질 시대 연대 표기 (단위 Ma/Ga 주의) |

**놓치기 쉬운 함정**:
- 최신 연구로 기존 수치가 갱신된 경우 (예: 공룡 크기 재추정)
- 대중 매체가 과장한 수치를 학술 근거처럼 인용하는 사례

---

### 역사
**서브카테고리 예**: 전쟁·정치·문화·인물

| 항목 | 확인 포인트 |
|---|---|
| 연도 | 사건 발생/시작/종료. **음력·양력** 또는 **역법 차이**(율리우스 vs 그레고리우스) |
| 인물 | 풀네임, 직책, 생몰년. 동명이인 주의 |
| 장소 | 국가·지역·지명. 당시 국명 vs 현재 국명 |
| 사건 순서·인과 | 시간 순서 정확한지, 원인–결과 관계가 과장 없는지 |
| 규모 | 사상자·병력·피해. 1차 사료 vs 후대 추정치 출처 명시 |
| 1차 vs 2차 사료 | 동시대 기록 vs 후대 정리본 구분. 인용 시 명시 |

**놓치기 쉬운 함정**:
- "영웅 서사"형 과장 (단일 인물 공로 vs 집단 기여)
- 전투 병력 수는 당대 기록이 과장된 경우 많음 → 현대 사학 추정치와 대조

---

### 게임 / 이스터에그·트리비아
**서브카테고리 예**: 숨겨진 기능, 개발 뒷이야기, 버그/글리치

| 항목 | 확인 포인트 |
|---|---|
| 게임명 | 풀 타이틀 + 부제 + 지역별 명칭 차이 |
| 발매 연도 | 지역별 발매일 차이. 초판 vs 리마스터 구분 |
| 개발사·퍼블리셔 | 당시 vs 현재 소유권. 합병·폐업 이력 |
| 플랫폼 | 오리지널 플랫폼, 해당 기능이 특정 플랫폼/버전 한정인지 |
| 기능 발견 연도 | 발매 연도와 발견 연도가 다를 수 있음 (수십 년 후 발견 사례 多) |
| 트리거 조건 | 정확한 재현 방법. 영상·공식 확인 자료 있는지 |
| 공식 vs 루머 | 개발자 인터뷰·공식 SNS 확인 vs 커뮤니티 추정만 있는 경우 구분 |

**놓치기 쉬운 함정**:
- Reddit·유튜브 루머가 사실처럼 퍼진 경우 (특히 Creepypasta 계열)
- 데이터마이닝으로 발굴된 것 vs 실제 게임에서 트리거되는 것 구분

---

### 야구
**서브카테고리 예**: 기록·사건·선수 일화

| 항목 | 확인 포인트 |
|---|---|
| 선수명 | 풀네임 + 등록명 (한국 선수 한자·영문 표기 확인) |
| 기록 연도·경기 | 정확한 시즌·날짜·경기번호. 연장전·서스펜디드 경기 특수 사례 |
| 당시 구단 | 트레이드 시점 기준. 현재 소속 구단과 혼동 금지 |
| 수치 | 타율·승수·홈런·ERA 등. 소수점 표기 관례 (.357 vs 0.357) |
| 리그·포지션 | 해당 시즌 소속 리그, 주 포지션. 포지션 변경 이력 |
| 공식 vs 추정 | KBO/MLB 공식 기록 vs 일화·추정치 구분 |

**놓치기 쉬운 함정**:
- 비공식 기록·세이버메트릭 vs 공식 기록 혼용
- "역대 최초·최고" 류 주장은 범위(리그·국가·전 세계·현대 야구 기준) 명시

## 보고 포맷 (결과 표)

```
| 주장 | 검증 결과 | 출처 | 신뢰도 | 판정 |
|---|---|---|---|---|
| 바거 293는 14,200톤 | 14,200 metric tons 확인 | Wikipedia EN / TAKRAF 공식 | S | ✅ OK |
| 제조사 ThyssenKrupp | 실제 제조사는 TAKRAF (구 VEB TAKRAF) | Wikipedia EN | A | ❌ 수정 필요 |
| 지면 압력 17 N/cm² | 출처 간 수치 편차 (17~22) | Wiki EN·독문 자료 | B | ⚠️ 보수적 범위 표기 권장 |
```

**판정 기준**:
- ✅ OK: 복수 고신뢰 출처 일치
- ❌ 수정 필요: 확실한 오류
- ⚠️ 보수적 표기: 출처 간 편차 → 범위·"약" 같은 완충 표현 권장
- ❓ 추가 확인: 1차 출처 부재, 대본 반영 보류
